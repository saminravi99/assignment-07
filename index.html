JavaScript is a client-side scripting language that primarily aids in creating interactive websites. JavaScript is also known as a programming language with a single thread. It features a single call stack and only executes one function at a time. When you start a JavaScript program, an Execution Context is produced. This execution context allocates memory to all of the variables and functions in that program. When JavaScript performs a function, the execution context moves from Global Execution Context to Local Execution Context and then back again. JavaScript uses the Call Stack to keep track of this switching. After returning to the Global Execution Context, the Call Stack is cleaned, restoring the program to its starting state.

Javascript is a synchronous programming language with a single thread that executes a single function at a time in a synchronous way. However, Javascript may alsoÂ do asynchronous behavior in particular instances, and the Event Loop is the principal concept underlying Javascript's asynchronous programming. Call Stack keeps track of all the synchronous and asynchronous functions of a program. The function gets popped off of the stack when it has completed its execution. The Event Queue is in charge of dispatching new functions to the stack for execution. It keeps track of the proper order in which all functions should be executed. When an asynchronous function, such as the setTimeout Method, fetch Method, or setTimeInterval Method, is called, it is routed to the built-in browser API, which begins execution of the function in its own single-threaded process, with the browser API acting as a separate thread. The Event Loop continuously checks to see whether the call stack is empty. If it is empty, new functions from the event queue are added. If it isn't, the current function call is executed.

The expiration time is the only significant difference between Local Storage and Session Storage.Persistent storage is provided via Local Storage, which may be accessed even when the browser is closed. Because of this, the storage is preserved even if the domain or browser is opened in a new tab or restarted. There's no time limit attached. Whereas, Session Storage is cleared out after closing the page session. A page session is effective as long as the tab or window is active. So when a new tab or window is opened, the browser effectively creates a new session and stores data for the duration of that session.
